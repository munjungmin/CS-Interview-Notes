### 입출력 장치

#### **❓ 보조기억장치와 주기억장치는 무엇이 다른가요?**
보조기억장치는 주기억장치보다 속도는 느리지만 데이터를 영구히 보관할 수 있습니다.  
주기억 장치는 CPU가 작업 중 사용하는 데이터와 명령어를 임시로 저장하며, 컴퓨터가 꺼지면 데이터가 사라집니다.  
주기억 장치로는 RAM이 있고 보조기억장치로는 HDD, 플래시 메모리 등이 있습니다.  
<br>


#### **❓ 보조기억장치는 왜 주기억장치는 느린가요?**
보조기억장치는 데이터를 저장하기 위해 기계적 동작이나 복잡한 전자적 제어 과정을 거치기 때문에 주기억장치보다 접근 속도가 느립니다.  
HDD의 경우 디스크 회전과 헤드 이동 같은 물리적 지연이 발생하고, SSD는 RAM과 같은 반도체 기반이지만 RAM은 CPU와 직접 데이터를 주고받는 방식이므로, 훨씬 빠르게 데이터에 접근이 가능합니다.  
<br>


#### **❓SSD와 HDD의 구조적 차이는 무엇인가요?**
HDD는 여러 장의 플래터를 회전시키고, 디스크암에 달린 헤드가 트랙을 이동하며 데이터를 자기장을 이용해 읽고 쓰는 기계식 저장장치입니다.  
반면 SSD는 플래터나 헤드 같은 움직이는 부품이 없고, NAND 플래시 메모리 셀에 전하로 데이터를 저장하는 반도체 기반 저장장치입니다.  
그래서 HDD는 탐색/회전 지연 같은 물리적 지연이 크고, SSD는 그런 지연은 없지만 플래시 특성상 페이지/블록 단위 제약과 컨트롤러 관리 오버헤드가 존재합니다.

*ssd는 읽기/쓰기는 페이지 단위(4kb~)로, 삭제는 블록 단위(수백kb~)로 수행함.  
이유는 셀 하나만 깔끔하게 지우기 어렵고, 높은 전압을 써서 한번에 지우는게 효율적이기 때문*<br>
<br>
#### **❓가상 메모리는 이 둘을 어떻게 연결하나요?**
가상 메모리는 페이징(Paging) 기법을 통해 보조기억장치와 주기억장치를 연결합니다.
프로세스가 사용하는 가상 주소를 페이지 테이블을 이용해 물리 메모리의 실제 주소로 변환하는데, 만약 필요한 페이지가 RAM에 없으면 페이지 폴트가 발생하고, 운영체제가 디스크(스왑 영역)에서 해당 페이지를 RAM으로 로드합니다. 필요시 기존 페이지를 디스크로 내보내는 스왑 아웃도 함께 일어납니다.  
<br>

#### **❓HDD의 데이터 접근 시간에 대해 설명해주세요**
HDD의 데이터 접근 시간은 탐색시간, 회전 지연 시간, 전송 시간을 모두 더한 값을 말합니다.  
먼저 탐색 시간은 디스크암을 움직여 특정 트랙으로 헤드를 이동시키는 시간입니다. 회전 지연 시간은 디스크가 회전하여 헤더가 목표 섹터 위에 위치하는데 걸리는 시간입니다. 마지막으로 전송 시간은 헤드가 실제 데이터를 읽고 쓰는 데 걸리는 시간입니다.  
이 중 가장 오래 걸리는 작업이 탐색 시간이기 때문에 디스크 암의 이동거리를 최소화할 수 있도록 데이터 접근 순서를 스케줄링 해야 합니다.  
<br>

#### **❓플래시메모리에 대해 설명해주세요**
플래시 메모리는 전자적인 방식으로 데이터를 저장하는 보조 기억 장치로 USB, SSD가 있습니다.  
하드디스크와 달리 움직이는 부품이 없어 외부 충격에 강합니다. 셀 단위로 데이터를 저장하는데 데이터를 여러 번 쓰고 지우면 셀이 마모되기 때문에 쓰기 수명이 존재합니다. 컨트롤러라는 부품이 데이터의 읽기, 쓰기, 삭제 작업을 제어하고 셀의 수명을 관리합니다.  
<br>


#### **❓RAID는 무엇인가요?**
RAID는 여러 개의 기억장치를 결합하여 속도, 안정성, 데이터 보호를 개선하는 기술입니다. 
여러 개의 디스크가 묶여 하나의 논리적인 디스크로 작동하게 됩니다. 데이터를 분산 저장하거나 복제하여 단일 디스크 고장의 위험을 줄일 수 있습니다. 

*RAID 수준
RAID 0(스트라이핑) : 여러 개의 디스크에 데이터 저장  
RAID 1(미러링) : 데이터를 그대로 백업한 디스크 사용  
RAID 2(섹터 스트라이핑): 일부 디스크에 ECC 정보 저장  
RAID 3(섹터 스트라이핑): 한 디스크에 패리티 저장  
RAID 4(블록 스트라이핑): 하나의 패리티 디스크 사용  
RAID 5(블록 스트라이핑): 여러 디스크에 패리티를 분산 저장  
RAID 6(블록 스트라이핑) : 이중 패리티로 분산 저장   
RAID10(1+0): 미러링 후 스트라이핑*

*현재는 0, 1, 5, 10을 많이 사용함*  
<br>

#### **❓RAID는 0, 1, 5, 10의 차이점을 설명해주세요**
RAID 0은 스트라이핑 방식으로 데이터를 여러 디스크에 분산 저장합니다. 최소 2개의 디스크가 필요하며, 읽기/쓰기 속도가 가장 빠르지만 디스크 하나만 고장나도 모든 데이터가 손실됩니다.  
RAID 1은 미러링 방식으로 2개 이상의 디스크에 같은 데이터를 저장합니다. 디스크 하나가 고장나도 데이터가 안전하지만, 저장 공간은 50%만 사용 가능합니다.  
RAID 5는 스트라이핑과 패리티를 결합한 방식입니다. 최소 3개의 디스크가 필요하며, 패리티 정보를 분산 저장하여 디스크 1개 장애까지 복구가 가능합니다. 패리티 계산때문에 쓰기 속도가 조금 느립니다.  
RAID 10은 RAID 1과 RAID 0을 결합한 방식으로, 먼저 미러링하고 스트라이핑합니다. 최소 4개의 디스크가 필요하며, 속도와 안정성이 모두 뛰어납니다.
