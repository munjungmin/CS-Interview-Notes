### 데이터베이스

**❓데이터베이스는 무엇이며 어떤 특징이 있나요?**

데이터베이스는 여러 사람이 공유할 목적으로 만든 데이터 저장소입니다.
실시간 접근성, 동시성, 무결성, 데이터 중복 최소화, 보안성, 독립성 등의 특징을 가집니다.
<br><br><br>

**❓데이터베이스 모델링 단계를 간략히 설명해주세요**

모델링은 크게 개념적 모델링, 논리적 모델링, 물리적 모델링으로 나뉩니다.
개념적 모델링은 요구사항을 분석하여 저장할 데이터를 정리하고 개념적 데이터 모델로 표현하는 단계입니다. 주로 ERD가 많이 사용됩니다.
논리적 모델링은 ERD를 릴레이션 스키마로 구체화하는 작업을 수행합니다. 이 단계에서는 테이블과 속성, 기본키, 외래키 같은 관계를 설정하고 데이터를 정규화합니다.
물리적 모델링은 논리적 모델링을 기반으로 DBMS로 구현 가능한 물리적 구조를 설계하는 단계로 물리적 스키마를 생성합니다.
<br><br><br>

### DBMS

**❓DBMS는 무엇인가요?**

데이터베이스 관리 시스템으로, 데이터베이스 내 데이터에 접근하도록 도와주는 시스템입니다. 데이터의 저장, 수정, 검색, 삭제 기능 등을 제공합니다. 대표적인 DBMS로 MySQL, 오라클 등이 있습니다.
<br><br><br>

**❓DBMS가 제공하는 무결성 기능에는 무엇이 있나요?**

DBMS는 개체 무결성, 참조 무결성, 도메인 무결성, 고유 무결성을 제공합니다. 개체 무결성(기본키 제약)은 기본키가 NULL이거나 중복될 수 없음을 보장하고, 참조 무결성은 외래키가 참조 테이블에 존재하는 값만 가질 수 있도록 보장합니다. 도메인 무결성은 컬럼에 허용된 타입과 값만 입력되도록 제한하고, 고유 무결성(UNIQUE 제약)은 특정 컬럼의 값이 중복되지 않음을 보장합니다.
<br><br><br>

### SQL

**❓DBMS 언어에는 어떤 종류가 있나요?**

크게 3가지로, 데이터 정의어(DDL), 조작어(DML), 제어어(DCL)가 있습니다.
정의어는 테이블이나 인덱스를 생성, 수정, 삭제할 때 사용하며 CREATE, ALTER, DROP, TRUNCATE 등이 있습니다. 조작어는 데이터를 생성, 조회, 수정, 삭제하는 언어로 SELECT, INSERT, UPDATE, DELETE가 있습니다. 제어어는 DB 접근 권한을 설정하는데 사용하며 GRANT, REVOKE 등이 있습니다.
<br><br><br>

**❓TRUNCATE와 DELETE의 차이를 설명해주세요**

TRUNCATE는 테이블의 전체 데이터를 한번에 삭제해서 빠르고, DDL이라 즉시 자동 커밋이 되기 때문에 롤백이 불가능합니다. 반면 DELETE는 WHERE절로 삭제할 행을 지정할 수 있고, 행을 하나씩 순회하며 삭제하기 때문에 느립니다. 또 삭제 내역을 트랜잭션 로그에 기록하여 롤백이 가능합니다.
<br><br><br>

**❓GRANT와 REVOKE는 어떤 상황에서 사용하나요?**

GRANT는 사용자에게 권한을 부여하고, REVOKE는 부여한 권한을 회수할 때 사용하는 DCL 명령어입니다. 권한의 종류로는 테이블 조회, 삽입, 수정, 삭제 등 객체 권한과, 테이블 생성, 삭제 등 시스템 권한이 있습니다.
<br><br><br>

**❓DELETE와 DROP의 차이는 무엇인가요?**

DELETE는 테이블에서 특정 행을 삭제하는 DML이고, DROP은 테이블 자체를 완전히 제거하는 DDL입니다. 따라서 DELETE 후에는 테이블이 남아있어 새로운 데이터를 삽입할 수 있지만, DROP 후에는 테이블 자체가 사라지므로 다시 사용하려면 CREATE로 재생성해야 합니다.
<br><br><br>

### Key

**❓키(Key)의 특징과 다양한 키의 종류에 대해 설명해주세요**

키는 유일성과 최소성을 가집니다. 유일성은 특정 키 값은 테이블 내에서 중복될 수 없다는 것이고, 최소성은 키가 최소한의 속성으로만 구성되어야 한다는 것입니다. 키의 종류에는 기본키, 외래키, 후보키, 대체키, 슈퍼키 등이 있습니다.
슈퍼키는 최소성은 고려하지 않고 유일성만 만족하는 키이고, 후보키는 유일성과 최소성을 모두 만족하는 키입니다. 기본키는 후보키 중 대표로 선택된 키로, NULL과 중복을 허용하지 않으며, 대체키는 후보키 중 기본키로 선택되지 않은 키입니다. 외래키는 다른 테이블의 기본키를 참조하여 테이블 간의 관계를 나타내는 키를 말합니다.
<br><br><br>
